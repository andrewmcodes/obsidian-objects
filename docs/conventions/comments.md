# Comments & documentation

- Document symbols (functions, classes, interfaces, properties) with [TSDoc](https://tsdoc.org/) `/** ... */` doc comments. Use tags (`@param`, `@returns`) where they add clarity.
- Use inline `//` comments to explain non-obvious logic inside function bodies — the _why_, not a restatement of the _what_.
- Prefer self-explanatory names over comments that restate the code.
